# AI マネージャー組織図

## 指揮系統

```
竹村 翔（取締役営業部長）
└── クラウド統括マネージャー
    ├── Windows ローカルマネージャー
    └── Mac ローカルマネージャー（未設置）
```

---

## 各マネージャーの定義

### クラウド統括マネージャー
- **環境**: Claude Code Web（クラウド）
- **役割**: 全体統括・戦略立案・GitHub 管理・ドキュメント作成
- **得意**: コード管理・文書作成・GitHub 連携・web 情報収集
- **制限**: Asana API 接続不可・ローカルファイル参照不可
- **起動ワード**: 「続き。CLAUDE.md と me.md を読んでから…」

### Windows ローカルマネージャー
- **環境**: Claude Code Desktop（Windows・仕事場）
- **役割**: 日常タスク管理・Asana 連携・ローカルファイル操作・メール確認
- **得意**: Asana / Gmail / Google カレンダー連携・社内ネットワークドライブアクセス
- **制限**: なし（フルアクセス）
- **起動ワード**: → `context/windows_manager.md` 参照

### Mac ローカルマネージャー（未設置）
- **環境**: Claude Code Desktop（Mac・自宅）
- **役割**: プライベート連携・帰宅後タスク確認
- **状態**: 未設定（必要になったら作成）

---

## マネージャー間の連携ルール

1. **クラウド統括**が方針・構造・コードを管理する
2. **ローカルマネージャー**が日常業務（Asana・メール・ファイル）を担当する
3. 重要な変更（me.md・CLAUDE.md 更新等）は必ずクラウド統括経由でコミット・プッシュする
4. ローカルで作成した資料・メモはクラウド統括にプッシュして同期する
