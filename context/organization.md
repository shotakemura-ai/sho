# AI マネージャー組織図

## 指揮系統

```
竹村 翔（取締役営業部長）
├── Windows 統括マネージャー  ← メイン（仕事場）
├── Mac 統括マネージャー      ← メイン（自宅・未設置）
└── クラウドマネージャー      ← 補助（GitHub・コード・Web）
```

---

## 各マネージャーの定義

### Windows 統括マネージャー
- **環境**: Claude Code Desktop（Windows・仕事場）
- **位置づけ**: メイン。日常業務のほぼすべてを担当
- **得意**: Asana / Gmail / Google カレンダー連携・ネットワークドライブ・ローカルファイル操作・経費精算・帳票作成
- **制限**: なし（フルアクセス）
- **起動ワード**: `context/windows_manager.md` 参照

### Mac 統括マネージャー（未設置）
- **環境**: Claude Code Desktop（Mac・自宅）
- **位置づけ**: メイン（プライベート）。帰宅後・休日の作業
- **状態**: 未設定（必要になったら作成）

### クラウドマネージャー
- **環境**: Claude Code Web（クラウド）
- **位置づけ**: 補助。ローカルが使えない場面・GitHub 管理・コード作業に特化
- **得意**: GitHub 管理・ドキュメント作成・Web 調査・設定ファイル管理
- **制限**: Asana API 接続不可・ローカルファイル参照不可

---

## 使い分けガイド

| 作業内容 | 使うマネージャー |
|---------|----------------|
| Asana タスク確認・更新 | Windows / Mac |
| メール確認・返信 | Windows / Mac |
| カレンダー確認 | Windows / Mac |
| 社内ファイル操作 | Windows |
| 経費精算・帳票作成 | Windows |
| GitHub / コード管理 | クラウド |
| Web 調査・情報収集 | クラウド または Windows / Mac |
| 設定ファイル・CLAUDE.md 更新 | クラウド（git でコミット・プッシュ） |

---

## 共通ルール

1. 各マネージャーは対等。クラウドはローカルの上位ではない
2. **me.md・CLAUDE.md の変更**は必ず git コミット・プッシュして全マネージャーと同期する
3. 重要な資料・メモは git にコミットして共有する
4. 各マネージャーはセッション開始時に `git pull` で最新状態を取得する
